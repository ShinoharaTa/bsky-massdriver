# 機能棚卸し（現行 + 追加予定）

## 1. 認証・セッション

- Bluesky ログイン（identifier + app password）
- セッション永続化（`localStorage` の `sess`）
- セッション復元（起動時）
- ログアウト（セッション削除 + `/login` 遷移）

## 2. 投稿機能（メイン）

- テキスト投稿（Bluesky API）
- `@mention` と URL を `RichText` で facet 化
- 投稿前に前後空白/改行をトリム
- 文字数カウンタ表示（`0/300`）

## 3. 投稿補助

- テンプレート追加（現在入力を保存）
- テンプレート一覧表示
- テンプレート適用（テキストエリアへ反映）
- テンプレート削除
- テンプレートの永続化（`localStorage` の `texts`）

## 4. 共有機能

- `intent` 付き投稿URLを生成
- クリップボードへコピー
- `?intent=...` から初期テキストへ復元

## 5. 通知・状態表示

- ローディングオーバーレイ
- メッセージオーバーレイ（約2秒表示）
  - 例: `Post completed!`, `Post failed.`, `Please Login.`

## 6. 画面構成

- `/login` : ログイン画面
- `/` : 投稿画面（認証必須）
- `/information` : 使い方/説明

## 7. ナビゲーション・共通UI

- ヘッダー（ロゴ + ユーザーアイコン）
- ユーザーメニュー（アバター Popover からログアウト）
- フッターリンク（How To / GitHub / Support）

## 8. 現時点のUX課題（再設計で解く対象）

- 主要操作の優先度が視覚的に弱い
- 投稿補助機能（テンプレート/コピー）の配置が散っている
- ログイン→投稿までの導線が情報過多
- 成功/失敗通知の視認性が一定でない

## 9. 追加予定機能（今回の要件）

### 9-1. 複数アカウントログイン

- 複数の Bluesky アカウントを保存・管理
- アカウントごとのセッション状態を保持
- 追加・削除・再ログイン導線を用意

### 9-2. 投稿対象アカウントの同時選択

- 投稿時に送信先アカウントを複数選択可能
- 「全選択」「個別選択」「前回選択の保持」を検討
- 送信結果をアカウント単位で表示（成功/失敗）

### 9-3. 画像投稿機能

- ツールバーの「Image」ボタンからファイル選択（ドロップゾーン不要）
- 選択後はテキストエリア直下にコンパクトなサムネイルを表示
- 最大 4 枚 / JPEG・PNG・WebP 対応
- 各サムネイルに個別削除ボタン

### 9-4. ハッシュタグ履歴

- 投稿テキスト内の `#tag` を RichText パース結果から自動抽出
- 使用したハッシュタグを履歴として `localStorage` に記録
- 補助UIやプリセット候補は設けない（ユーザーが本文に直接手打ちする）

## 10. 再設計の前提ルール（新機能対応）

- 既存の「最短投稿体験」を壊さない（入力→投稿は最短1操作）
- 高機能化しても、初見ユーザーには段階的に機能を見せる
- 画像添付はツールバーボタン経由（専用エリアを取らない）
- ハッシュタグは本文内で完結し、専用UIは持たない
- スマホで極力スクロールしないで使える情報密度を保つ
