# 画面再構成（ドラフト）

## ゴール

- 「最短で投稿できる」ことを主体験にする
- 補助機能は主操作を邪魔しない位置に整理する
- 学習コストを下げる（初見で迷わない）

## 画面責務

## A. ログイン画面 `/login`

- 目的: 認証の完了だけに集中
- 構成:
  - ブランドヘッダー
  - 説明文（短く）
  - Handle / App Password 入力
  - Primary: `Login`
  - Secondary: `Sign Up`, `How To`

## B. 投稿画面 `/`

- 目的: 投稿作成と送信（スマホでスクロールなしが理想）
- 構成:
  - ヘッダー（ロゴ / アバター群 / アカウント追加ボタン）
  - アカウントバー（インラインチップ、チェックで投稿対象を選択）
  - メインカード:
    - テキストエリア
    - 画像サムネイル列（添付時のみ表示、56px 正方形）
    - アクションバー: 文字数 | + Template | Copy URL | 🖼 Image | Lift Off!
  - テンプレート一覧（コンパクト行）
- ハッシュタグ: 本文内で `#tag` を手打ち → RichText 変換 → 履歴に自動記録

## C. 情報画面 `/information`

- 目的: 使い方と tips の補助
- 構成:
  - 3セクション（What / How / Tips）
  - 末尾 CTA `Use It`

## 追加で必要なUIブロック

1. `AccountBar`（投稿対象チップ選択 — インライン表示）
2. `ImageThumbs`（添付サムネイル列 — テキストエリア直下）
3. `PostResultModal`（アカウントごとの結果表示オーバーレイ）

## 再設計時の優先順位（新要件込み）

1. 投稿画面の情報密度を最小化（スマホ1画面完結）
2. アカウントセレクタのインラインチップ操作モデル確定
3. 画像投稿フロー: ツールバーボタン → ファイル選択 → サムネイル → 投稿
4. ハッシュタグは専用UIなし（本文手打ち → 自動抽出 → 履歴保存）
5. ボタン階層（Primary/Secondary/Danger）の統一
6. メッセージ表示の統一（成功/失敗/要ログイン）

## コンポーネント分割案

- `ComposerCard` : テキストエリア + 画像サムネイル + アクションバー
- `AccountBar` : インラインチップ（チェック付き）
- `ImageThumbs` : 添付済み画像の小サムネイル列
- `TemplateList` : 一覧 + 選択 + 削除
- `UserMenu` : アバター群 + アカウント追加
- `GlobalFeedback` : Loading / Toast
- `PostResultModal` : 投稿結果サマリー（オーバーレイ）
